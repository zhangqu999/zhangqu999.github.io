2、分离部署到Linux服务器

2.1、后端部署

2.1.1、打包

使用IDEA里的maven插件，跳过test，package打包

打包完成后，会发现target目录下多了两个jar包

其中.jar包含了pom.xml文件中的依赖，是可执行jar包，可以通过java -jar直接运行，一般部署时使用。

.jar.original是普通jar包，不包含依赖，不可以直接运行，但可以给别的项目使用。

2.1.2、在服务器里启动

使用xftp将本地jar包，传输服务器某个目录下，比如我是在/usr/local目录下

编写start.sh脚本

#!/bin/sh
RESOURCE_NAME=recruitsystem-1.0.0-SNAPSHOT.jar
 
tpid=`ps -ef|grep $RESOURCE_NAME|grep -v grep|grep -v kill|awk '{print $2}'`
if [ ${tpid} ]; then
echo 'Stop Process...'
kill -15 $tpid
fi
sleep 5
tpid=`ps -ef|grep $RESOURCE_NAME|grep -v grep|grep -v kill|awk '{print $2}'`
if [ ${tpid} ]; then
echo 'Kill Process!'
kill -9 $tpid
else
echo 'Stop Success!'
fi
 
tpid=`ps -ef|grep $RESOURCE_NAME|grep -v grep|grep -v kill|awk '{print $2}'`
if [ ${tpid} ]; then
    echo 'App is running.'
else
    echo 'App is NOT running.'
fi
 
rm -f tpid
nohup java -jar ./$RESOURCE_NAME &
echo $! > tpid
echo Start Success!

运行脚本 ./start.sh

至此，后端项目就部署成功了。

2.2、前端部署

2.2.1、打包

通过yarn build命令会在 dist/ 目录产生一个可用于生产环境的包，带有 JS/CSS/HTML 的压缩，和为更好的缓存而做的自动的 vendor chunk splitting。它的 chunk manifest 会内联在 HTML 里。

2.2.2、使用Nginx部署

2.2.2.1、Nginx安装

使用yum进行安装

yum -y install nginx

其中 -y 表示安装过程中提示选择全为yes

默认会安装在/etc/nginx目录下

此时直接通过service nginx start 命令启动的话，会启动失败，通过systemctl status nginx.service命令查看报错信息，会发现是第5行的错。

查看nginx.conf配置文件，发现第5行是 user nginx; 而我们系统里并没有nginx这个用户，因此会报错。

这里改成 user nobody就可以了。

然后再通过service nginx start 命令就可以启动成功了。

2.2.2.2、Nginx配置

修改nginx.conf文件

修改server配置如下：

    server &#123;
        listen       8080;
        server_name  121.4.20.119;
        root         /usr/share/nginx/html;
        # Load configuration files for the default server block.
        include /etc/nginx/default.d/*.conf;

        #重写nginx默认欢迎页面
        location / &#123;
         root /usr/local/dist;  #dist目录绝对路径
         index index.html;
         try_files $uri $uri/ /index.html;  #重定向
        &#125;

    #配置后端的接口，将访问域名/api的路径代理给我们后端的接口
    location /api &#123;
        proxy_pass http://121.4.20.119:3000;
    &#125;


        error_page 404 /404.html;
        location = /404.html &#123;
        &#125;

        error_page 500 502 503 504 /50x.html;
        location = /50x.html &#123;
        &#125;
    &#125;

然后通过 service nginx restart 命令重启nginx，就可以通过 http://121.4.20.119:8080/ 正常在线访问了。

3、debug记录

3.1、后端

3.1.1、spring-boot-maven-plugin not found

加上version坐标就可以了

3.2、前端

3.2.1、PowerShell yarn : 无法加载文件 C:\Users\Admin\AppData\Roaming\npm\yarn.ps1,因为在此系统因为在此系统上禁止运行脚本。

以管理员方式启动powershell，执行以下命令更改计算机执行策略

set-ExecutionPolicy RemoteSigned

是否要更改执行策略，选择Y

然后查看执行策略

get-ExecutionPolicy

再重新执行yarn命令就OK了。

3.2.2、Error: Node Sass version 5.0.0 is incompatible with ^4.0.0

将node-sass@5.0版本卸载

yarn remove node-sass

然后安装4.14.1版本

yarn add node-sass@4.14.1 --dev

3.2.3、npm的connect etimedout xx.xx.xx.xx:443

这个是我在将npm配置了taobao源之后仍然出现的问题，尝试用cnpm

  • 安装

    npm install -g cnpm

  • 使用

    cnpm install

3.2.4、Linux环境下安装nginx,启动报错 nginx：nginx: [emerg] getpwnam(“nginx”) failed

将nginx.conf第5行的user nginx; 改成 user nobody;

3.2.5、前端项目打包之后index.html会出现三个不消退的div提示信息

直接删掉index.html里这三个div即可。
