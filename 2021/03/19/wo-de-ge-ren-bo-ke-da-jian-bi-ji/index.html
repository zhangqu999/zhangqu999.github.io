1、搭建hexo博客，并部署到GitHub Pages

1.1、安装node

1.1.1、下载安装nodejs

http://nodejs.cn/download/ 选择Windows 64位安装包，安装一直next就好了

1.1.2、查看node和npm版本

node -v
npm -v

1.1.3、修改npm地址为淘宝镜像

npm config set registry http://registry.npm.taobao.org/

1.2、安装并使用hexo

1.2.1、安装hexo框架

npm install -g hexo-cli

会装在nodejs/node_global目录下（D:\Program Files\nodejs\node_global），然后查看hexo版本

hexo -v

1.2.2、初始化hexo博客

  • 在指定目录下创建myBlog空目录，用来存放个人hexo博客网站所有内容

    mkdir D:\myBlog

  • 进入上一步创建的myBlog目录

    d:    #切换到D盘
cd myBlog

  • 初始化hexo博客，会默认生成一篇helloworld博客

    hexo init

1.2.3、启动本地博客服务

hexo s

启动成功之后，就可以通过 http://localhost:4000/ 访问本地博客地址了

选项 描述
-p, --port 重设端口
-s, --static 只使用静态文件
-l, --log 启动日记记录，使用覆盖记录格式

1.2.4、创建新的文章

hexo n "我的个人博客搭建笔记-1"

在_post目录下就会生成 我的个人博客搭建笔记-1.md 文件

参数 描述
-p, --path 自定义新文章的路径
-r, --replace 如果存在同名文章，将其替换
-s, --slug 文章的 Slug，作为新文章的文件名和发布后的 URL

1.2.5、清理hexo

清除缓存文件 (db.json) 和已生成的静态文件 (public)。在某些情况（尤其是更换主题后），如果发现您对站点的更改无论如何也不生效，您可能需要运行该命令

hexo clean

1.2.6、生成静态文件

hexo g
选项 描述
-d, --deploy 文件生成后立即部署网站
-w, --watch 监视文件变动
-b, --bail 生成过程中如果发生任何未处理的异常则抛出异常
-f, --force 强制重新生成文件 Hexo 引入了差分机制，如果 public 目录存在，那么 hexo g 只会重新生成改动的文件。 使用该参数的效果接近 hexo clean && hexo generate
-c, --concurrency 最大同时生成文件的数量，默认无限制

1.3、部署到GitHub Pages

1.3.1、在GitHub新建一个空仓库

命名必须是YourGithubName.github.io

1.3.2、在myBlog目录下安装Git部署插件

npm install --save hexo-deployer-git

1.3.3、修改全局配置文件_config.yml

该文件位于你的博客根目录下，比如我的在D:\myBlog\_config.yml，修改里面的Deployment配置部分如下：

deploy:
  type:  git
  repo: https://github.com/zhangqu999/zhangqu999.github.io
  branch: master

​ repo后面是你刚刚在GitHub新建的空仓库的地址

1.3.4、部署到GitHub

hexo d

然后你就可以通过 https://YourGithubName.github.io/ 这个地址访问你的博客（GitHub默认必须通过HTTPS访问，HTTP访问会报404）

1.4、修改博客主题

1.4.1、yilia

  • 在myBlog目录下执行下面命令，将yilia主题克隆到myBlog/themes/yilia目录下

     git clone https://github.com/litten/hexo-theme-yilia.git themes/yilia

  • 修改hexo根目录（myBlog）下的 _config.yml 文件 

    theme: yilia

  • 依次执行下面hexo命令

    hexo clean    #修改主题，必须清理
hexo g        #c重新生成静态文件
hexo d    #部署到远程Github仓库

  • 查看GitHub Pages是否更新成功

    https://YourGithubName.github.io/ ，这个更新可能会比较慢，多强制刷新几次

1.5、在typora里配置图片上传服务PicGo

1.5.1、下载安装PicGo

https://github.com/Molunerfinn/PicGo/releases Windows下载exe文件，安装一直next就可以了

1.5.2、在typora里配置PicGo

1.5.3、配置PicGo

我用的是阿里云的OSS图床，配置下KeyID和KeySecret，存储空间名（bucket），存储区域和图片上传路径（一定要以/结尾）

1.5.4、验证是否配置成功

点下那个验证图片上传选项按钮，检验下是否配置成功

1.6、使用hexo-git-backup插件备份源文件

1.6.1、安装

如果你的hexo版本是2.x.x，执行下面命令：

npm install hexo-git-backup@0.0.91 --save

如果你的hexo版本是3.x.x及以上，执行下面命令：

npm install hexo-git-backup --save

1.6.2、更新

npm remove hexo-git-backup
npm install hexo-git-backup --save

1.6.3、配置

修改博客根目录（myBlog）下的_config.yml文件

backup:
    type: git
    repository:
       github: https://github.com/zhangqu999/zhangqu999.github.io,backup

上面GitHub地址后面用英文逗号隔开，然后加上你的备份分支名（提前在你的xxx.github.io仓库创建好backup分支）

1.6.4、使用

hexo b

4、换电脑写博客并同步

4.1、源文件推到GitHub

在GitHub新建一个仓库（public和private均可，xxx.github.io必须public），并把本地源文件推到该仓库里。

4.2、新电脑所需软件

  • Git
  • Nodejs

自行安装配置好。

4.3、安装hexo

npm install hexo-cli -g

4.4、将GitHub源文件clone到新电脑

新建myBlog目录，在该目录下执行：

git clone https://github.com/zhangqu999/hexo-matery-source.git

4.5、安装node_modules依赖

  • 将npm改为淘宝源
  • 下载package.json里的npm依赖包
npm install

4.6、配置下git config

配置git全局邮箱，用户名和SSR代理

git config --global user.email "zhang1842279380@gmail.com"
git config --global user.name "zhangqu999"
git config --global https.proxy http://127.0.0.1:1080
git config --global http.proxy http://127.0.0.1:1080

4.7、测试

生成静态文件并部署到GitHub远端（xxx.github.io）

hexo d -g

4.8、备份源文件

新建一个push.sh脚本，用来快速将本地源文件更新push到源文件仓库。

if [ ! -n "$1" ]; 
then
    msg="small update"
else
    msg="$1"
fi 

git add .  
git commit -m "$msg"  
git push

通过./push.sh执行脚本，即可快速push到远端。
